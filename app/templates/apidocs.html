API documentation

Currently, the API has one endpoint: POST https://yalies.io/api/students. The endpoint returns a JSON list of students. It accepts a JSON-formatted body containing the properties query, filters, and page (which may all be omitted). query represents an arbitrary string to search by, which will search across most fields of student data. filters is a dictionary mapping column names to lists of acceptable values. page represents the number of the page of students to get, which may allow you to improve request efficiency if you do not need to fetch every student at once. For example:

{
    "query": "Joe",
    "filters": {
        "college": [
            "Grace Hopper",
            "Davenport"
        ],
        "year": [2023, 2024],
        "floor": [3],
        "leave": [True]
    },
    "page": 1
}

This payload would fetch the first page of students in Grace Hopper or Davenport college, in the classes of 2023 or 2024, who live on floor 3 of their residential building, are on a leave of absence, and have a name, address, or other property that contains the search term "Joe".

The following filters are supported: college, year, major, building_code, entryway, floor, suite, room, state, and leave.

Authentication

You must pass in the header Authorization, with a value of Bearer _______ (replacing the underscores with the token obtained from the top of this page).
